Beam-project

www.beam-project.com

About

Beam is an open source, cross platform application for displaying information about previous, current and next song in the playlist. With the use of Cortina-rules (see below), Beam can detect the next tanda and display information about it. Beam displays the information from the ID3-tag in the music files. These tags contain information such as Artist, Title, Album, etc. These tags are possible to modify to help Beam display the correct information.

Platforms and support

Beam supports various media-players Windows, Mac and Linux, it is possible to change the configuration while playing/displaying information. All supported players are listed under each platform below.

Windows 7

  • Itunes
  • FooBar2000

MacOSX

  • None

Linux

  • Audacious

Fullscreen

You can enter fullscreen on the display that Beam is currently on by either double-clicking on the background, F11 (or for Windows, Ctrl+F).

Configuration

All settings for Beam are accessed in the interface, click File -> Settings.

Basic Settings

In basic settings you choose:

  • Media player - Which media player to extract from. The available modules are shown in the drop down list.
  • Background image - The background on which the text is displayed, native resolution (the size of the screen) or higher is recommended.
  • Update time - How often Beam shall extract information from the music player. Short time gives faster response when changing songs, however, this will increase the load of the system. Default is 4000mSec (4Sec).

Display

Beam can display the following fields from ID3-tags

  • Artist, Album, Title, Genre, Comment, Composer and Year.

There are also two fields not present in the ID3-tag which are created in the application, these are Singer and NextTanda. These fields are empty by default and must be filled with information, see Cortinas and Rules.

The structure of NextTanda is different as all information is contained in this. For NextTanda[X] you find the same sequence as mentioned above, starting from X=0; Artist, X=1; Album, X=2; Title, etc

It is possible to specify which song to display this information from, starting with [0] for previous, [1] for current, [2] for next, etc.

Example:        
  • Title[0] will show previous song’s title.
  • Artist[2] will show next song’s artist.
  • Singer[1] displays the current singer.

Cortinas and Rules

There are 3 types of rules:

  • Set - Copies one field to another
  • Cortina - Defines what is, or is not a cortina
  • Parse - Parse a certain field in the ID3-tag

Set is useful if you want to move information from one field to another, for example if you store the singer in comment;

Example:
  • Set - Comment - Singer

Cortina will define a cortina based on information matching something in your library, this can be done by either inclusion (is) or exclusion (isNot). All cortina rules are added and applied in sequence.

Example:        
  • Cortina - is - Genre - Cortina
  • Cortina - isNot - Genre - Tango, Vals, Milonga
  • Cortina - is - Artist - Cortinas

Parse shall be used to determine differences in the naming scheme in your archive. For example, if you have - ( ) or other signs you want to exclude or split.

Example: I have an artist called Various artists where songs have the title
  • ARTIST - TITLE
  • Parse - Title - ‘-’ - Artist - Title
  • This will take the first part of Title and copy to Artist, and the second
  • part and copy to Title.

Note:

All rules are executed sequentially. This means in practice for these rules:

  1. Set - Comment - Singer
  2. Parse - Artist - ‘-’ - Artist - Singer

If we have Artist: Carlos Di Sarli - Alberto Podestá - Rule 1 will ALWAYS copy whatever is in the Comment to Singer, and if there are a dash, then the second rule will overwrite Singer with Alberto Podestá. However, if we change the order of the rules:

  1. Parse - Artist - ‘-’ - Artist - Singer
  2. Set - Comment - Singer

Would only remove the Singer from Artist, as we will ALWAYS overwrite Singer in the end. This is useful if we want to remove parts of the ID3-field.
